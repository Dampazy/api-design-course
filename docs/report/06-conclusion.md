Итоговый тест реализован с 7 вопросами (превышает минимальные требования
задания) — вопросы намеренно не дублируют 18 практических заданий, чтобы
проверять перенос знаний, а не запоминание конкретных формулировок.

## Технологический стек

- Backend: Python 3.9, FastAPI, SQLAlchemy 2.0, Pydantic 2, SQLite, pytest
- Frontend: React 18, React Router 6, Vite 5, react-markdown, Playwright

## Ограничения текущей реализации (осознанные, для учебного масштаба)

- Аутентификация упрощена до анонимного `session_id` — нет регистрации,
  входа, восстановления доступа. Достаточно для демонстрации персонального
  прогресса, но не готово к реальному multi-device использованию одним
  пользователем (session_id привязан к конкретному браузеру/устройству
  через `localStorage`).
- Итоговый тест не ограничен по времени и допускает неограниченное число
  попыток — для реального экзамена потребовалась бы отдельная логика
  разовой/ограниченной по времени попытки.
- Метрика времени ответа считается на клиенте и не учитывает переключение
  вкладок/паузы пользователя.

## Возможные направления развития

- Полноценная аутентификация (email/пароль или OAuth2) вместо анонимных
  сессий — для сохранения прогресса между устройствами.
- Ограничение числа попыток и времени на итоговый тест.
- Экспорт прогресса/сертификата о прохождении курса в PDF.
- Админ-интерфейс для редактирования контента курса без правки Python-файлов
  напрямую (сейчас контент правится в `seed/theory_data.py`/`tasks_data.py`
  и заливается через `python -m app.seed.seed`).

## Сдача результатов

Код, методические материалы (`docs/`) и данный отчёт передаются
руководителю практики. Защита — демонстрация работающего веб-приложения
(backend + frontend, запущенные локально по инструкции
[SETUP.md](../SETUP.md)) с прохождением полного пользовательского пути:
теория → практика с автопроверкой → итоговый тест → страница прогресса.
