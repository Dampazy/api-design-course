# 1. Обзор существующих платформ автопроверки

Перед проектированием собственного модуля проверки был проведён обзор
подходов, используемых существующими образовательными и тренировочными
платформами.

## Stepik

Универсальная образовательная платформа с широким набором типов заданий:
`choice` (одиночный/множественный выбор), `string`/`number` (текстовый
ответ с точным или regex-сравнением), `sorting` (упорядочивание),
`matching` (сопоставление пар), `code` (проверка через запуск
пользовательского кода и сравнение stdout с эталоном). Проверка
выполняется на сервере синхронно для простых типов и асинхронно (в очереди)
для `code`-заданий, требующих запуска в изолированном окружении (sandbox).

## Coursera

Делает акцент на `quiz` (multiple choice/checkbox) для теоретических
модулей и на `programming assignment` — автоматическую проверку
запускаемого кода через тестовый harness (аналог unit-тестов,
предоставляемых автором курса), с опциональным peer review для заданий,
которые сложно проверить автоматически.

## LeetCode / HackerRank

Специализируются на code-заданиях: пользовательский код компилируется и
запускается против набора скрытых тестовых случаев (test cases),
результат — бинарный (все тесты прошли / не все) либо с частичным
результатом (X из N тестов). Требует полноценной песочницы выполнения
кода — существенно более тяжёлая инфраструктура, чем нужно для целей
данного курса (тема курса — принципы проектирования API, а не
программирование как таковое).

## Codewars

Похож на LeetCode/HackerRank (code kata с автотестами), но добавляет
социальный слой: ранги (kyu), обсуждение решений после прохождения,
альтернативные решения других пользователей.

## Вывод для проектируемой платформы

Для темы «Проектирование API» задания преимущественно не требуют
выполнения произвольного кода — это концептуальные вопросы (выбор
HTTP-метода/кода ответа, распознавание нарушений конвенций, упорядочивание
шагов протокола, сопоставление терминов). Это позволило спроектировать
модуль автопроверки **без песочницы выполнения кода** (в отличие от
LeetCode/HackerRank/Codewars) — детерминированное сравнение структур
данных (choice-индексы, нормализованные строки, последовательности,
JSON-объекты), что проще, быстрее и безопаснее в реализации, при этом
покрывает 5 разных форматов заданий (см. [02-task-formats-comparison.md](02-task-formats-comparison.md)).
