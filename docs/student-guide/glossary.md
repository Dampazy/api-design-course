# Глоссарий

- **REST (Representational State Transfer)** — архитектурный стиль проектирования сетевых API вокруг ресурсов, идентифицируемых URI.
- **Ресурс** — сущность предметной области (пользователь, заказ), доступная по URI.
- **Safe-метод** — HTTP-метод, не изменяющий состояние сервера (GET, HEAD, OPTIONS).
- **Idempotent-метод (идемпотентный)** — метод, повторное выполнение которого приводит к тому же состоянию сервера, что и однократное (GET, PUT, DELETE).
- **RFC 7807 Problem Details** — стандартный формат JSON-тела ошибки HTTP-ответа с полями `type`, `title`, `status`, `detail`, `instance`.
- **Breaking change** — изменение API, ломающее совместимость с существующими клиентами (удаление или изменение семантики поля/эндпоинта).
- **Tolerant reader** — практика, при которой клиент игнорирует незнакомые поля в ответе API вместо падения с ошибкой.
- **AuthN (Authentication)** — подтверждение личности субъекта запроса.
- **AuthZ (Authorization)** — проверка прав субъекта на конкретное действие.
- **OAuth 2.0** — протокол делегированной авторизации без передачи пароля пользователя приложению.
- **Authorization Code Flow** — основной сценарий OAuth2: получение authorization code через редирект пользователя, затем обмен code на access token.
- **JWT (JSON Web Token)** — самодостаточный токен вида `header.payload.signature` с проверяемой подписью.
- **Scope (скоуп)** — гранула прав доступа, привязанная к токену (например `orders:read`).
- **Offset-based пагинация** — постраничная навигация через `limit`/`offset`.
- **Cursor-based пагинация** — навигация через непрозрачный курсор, кодирующий позицию последнего элемента.
- **Idempotency-Key** — заголовок с уникальным ключом попытки операции, защищающий POST-запросы от случайного дублирования при повторной отправке.
- **Rate limiting** — ограничение частоты запросов от одного клиента за окно времени.
- **HATEOAS (Hypermedia As The Engine Of Application State)** — принцип, при котором ответ API содержит ссылки на доступные из текущего состояния действия.
- **OpenAPI / Swagger** — машиночитаемый формат описания REST API, на основе которого генерируется интерактивная документация (Swagger UI).
- **PKCE (Proof Key for Code Exchange)** — расширение OAuth2 (RFC 7636), позволяющее безопасно использовать Authorization Code Flow без хранения `client_secret` — для мобильных и SPA-приложений.
- **Sunset (заголовок, RFC 8594)** — HTTP-заголовок ответа, сообщающий дату отключения эндпоинта/версии API.
- **Webhook** — «обратный» вызов API: сервер сам отправляет POST-запрос клиенту при наступлении события; подлинность обычно проверяется криптографической подписью тела запроса.
