# Деплой на Render.com

Проект развёрнут как единый сервис: FastAPI отдаёт и API (`/api/*`), и
собранный React-фронтенд (всё остальное) — один процесс, один URL, без
CORS между фронтендом и бэкендом в проде. Конфигурация — в `render.yaml`
в корне репозитория (Render Blueprint).

## Репозиторий

Код: [github.com/Dampazy/api-design-course](https://github.com/Dampazy/api-design-course)

## Шаги подключения (один раз)

1. Зайти на [dashboard.render.com](https://dashboard.render.com) и войти
   через "Sign in with GitHub" (аккаунт Render не требует привязки карты
   для free-плана).
2. Нажать **New +** → **Blueprint**.
3. Разрешить Render доступ к репозиторию `api-design-course` (если
   спросит — выбрать "Only select repositories" и указать именно его).
4. Render найдёт `render.yaml` в корне и предложит создать сервис
   `api-design-course` (Python, план Free) с уже заполненными командами
   сборки и запуска — ничего вручную вводить не нужно.
5. Нажать **Apply** / **Create Web Service** — начнётся первый деплой.

## Что происходит при деплое

- `buildCommand`: собирает фронтенд (`npm install && npm run build` →
  `frontend/dist`) и ставит зависимости backend (`pip install -r requirements.txt`).
- `startCommand`: поднимает `uvicorn app.main:app` на порту, который
  передаёт Render через переменную `$PORT`.
- При старте FastAPI (`app/main.py`, обработчик `startup`) проверяет, пуста
  ли таблица `theory_blocks` — если да, автоматически прогоняет
  `app/seed/seed.py`. Это значит: после каждого fresh-деплоя курс
  наполняется контентом сам, без ручного шага.

Первый деплой занимает несколько минут (сборка npm + установка Python
зависимостей). После этого Render выдаёт постоянный URL вида
`https://api-design-course.onrender.com` — им можно поделиться с
руководителем практики в любой момент.

## Ограничения бесплатного плана (важно знать)

- Сервис "засыпает" после ~15 минут без запросов и просыпается ~30–50
  секунд на первый запрос после простоя — это нормально для бесплатного
  плана, не ошибка.
- Файловая система эфемерна: при каждом перезапуске/новом деплое
  SQLite-файл создаётся заново и наполняется через auto-seed на старте.
  Значит, **прогресс пользователя между простоями не сохраняется** — это
  приемлемо для демонстрационного учебного проекта (курс и задания на
  месте всегда, обнуляется только история попыток).

## Обновление после изменений в коде

Render Blueprint настроен на автодеплой: любой `git push` в ветку `main`
репозитория запускает новый деплой автоматически.

```bash
git add -A
git commit -m "описание изменений"
git push
```
